แบบฝึกท้ายบท

*ให้นักศึกษาเลือกระบบ Automation/Integration Platform as a Service (iPaaS) หรือ Low-code Platform อื่นๆ มา 1 ตัว และเปรียบเทียบข้อดีข้อเสียกับ n8n โดยเน้นที่การใช้งานแบบ Local
ตอบ Node-RED ใช้งาน Local ได้เบาและเร็วกว่า n8n แต่ n8n เหมาะกับ workflow ที่ซับซ้อนและจัดการ API ได้ง่ายกว่ามาก.
-------------------------------------------------------------------------------------------------------
*ให้นักศึกษาเปรียบเทียบการใช้งาน n8n แบบ Local และ Cloud และการติดตั้งแบบ Local แบบ node.js และ Docker เหมาะสมกับสถานการณ์ใด
ตอบ- Cloud ความสะดวก
-Local (Docker) ระบบเสถียร ไม่อยากยุ่งกับโค้ด
-Local (Node.js) ปรับแต่งลึกและเรียนรู้เชิงเทคนิค
-----------------------------------------------------------------------------------------------------
*ให้นักศึกษาอธิบายหน้าที่ของ Node แต่ละประเภทของ n8n พร้อมยกตัวอย่าง ประเภทละ 3 ตัวอย่าง 
ตอบ -Trigger Node ใช้เริ่ม workflow เช่น Webhook, Cron (ทำงานตามเวลา), Email Trigger (เริ่มเมื่อมีเมลเข้า)
-Action Node ใช้ทำงานหลัก เช่น ส่งอีเมล (Gmail), เขียนข้อมูลลง Google Sheets, อัปเดตข้อมูลใน Notion
-Logic Node ใช้ควบคุมทิศทาง เช่น IF (เช็คเงื่อนไข), Switch (เลือกตามค่า), Merge (รวมข้อมูล)
-Utility Node ใช้จัดการข้อมูล เช่น Set (ตั้งค่าข้อมูล), Function (เขียนโค้ดเอง), Code (เขียน logic ขั้นสูง)
-----------------------------------------------------------------------------------------------------
*ให้นักศึกษายกตัวอย่างการใช้งาน n8n จริง (พร้อมอ้างอิงแหล่ง/ที่มา) มา 3 กรณี พร้อมอธิบายว่า n8n ช่วยแก้ปัญหาใดในแต่ละกรณี
ตอบ n8n ช่วยลดงานมือ, ประหยัดเวลา และเชื่อมระบบอัตโนมัติ เช่น จากฟอร์ม → Sheets, ข่าว → Discord, งาน → Slack ได้ง่ายโดยไม่ต้องเขียนโค้ดเองทั้งหมด.
